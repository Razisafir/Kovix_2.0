## 3. Bugs Found During Launch

### Bug 1: NLS messages file missing
```
Error reading NLS messages file /home/z/my-project/kovix-rebuild/out/nls.messages.json: ENOENT
```
**Severity:** Low — this is a development-only issue. The NLS file is generated by the build pipeline for localized string loading. In dev mode with `VSCODE_DEV=1`, the fallback is to use the inline English strings. Does not affect functionality.

### Bug 2: X11 display connection failure in standard launch mode
```
ERROR:ui/ozone/platform/x11/ozone_platform_x11.cc:257] Missing X server or $DISPLAY
```
**Severity:** Environment-specific — this only occurs because the Xvfb setup lacks `xauth` for proper X11 authentication. On a real desktop or a CI with `xvfb-run` + `xauth` installed, this would not happen. The `--ozone-platform=headless` workaround works for smoke testing.

### Bug 3: `@vscode/spdlog` native module not built by default
**Severity:** Low — spdlog is a performance logging module. The app continues without it (falls back to console logging). It was manually rebuilt during this session.
